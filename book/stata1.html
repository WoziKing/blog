                    # 1 STATA简介 | STATA初步教程

                    # 1.1 什么是STATA？这玩意能做啥？

                      对于数据而言，它的体量可以是庞大的，比如我国14亿人口的数据，或者某些短视频平台精准投放的BigData。同时它的体量也可以比较小，小到一个省、一个市、一个区、一个街道、一个小区、一个单元楼，甚至小到个人。
                      因此，在生活中我们所接触到的数据无穷尽也。那么，用最小的资源对这些数据进行最高效的处理，便显得比较重要。
                      正所谓“工欲善其事，必先利其器”，好的处理软件会便于我们对数据进行处理。在这种情况下，Stata诞生了。

                      Stata是一套提供其使用者数据分析、数据管理以及绘制专业图表的完整及整合性统计软件。
                      Stata的体量较小，安装后的文件在1.50GB左右。我曾使用过SAS、MatLab，SAS的大小记得在3~5个GB左右，MatLab就大了，能有16GB。
                      因此，在体量上Stata具有明显的优势。

                      麻雀虽小，五脏俱全。Stata虽然体量小，但其所具有的功能不输SAS等数据处理分析软件。
                      可见Stata是一款比较小巧的数据处理分析软件，而在1.2中，你将会知道，它同时也是一款比较友善的软件。

                    # 1.2 新手友好型软件

                      大部分的同学可能都没有接触过较专业的数据分析处理软件，而Stata在这里便显示出它“新手友好型”的特点。

                    # 1.2.1 命令式输入

                      Stata不同于SAS、Matlab这种基于编程语言进行输入的软件，它的输入均为命令式输入。
                      “命令式输入”这是一个什么概念，以帮助指令为例：

                    命令使用

                    help [command]

                    关于中括号

                    用中括号[]括住的内容为选填内容

                      不难发现这种命令式输入“一气呵成”，只有单纯的命令，这就十分方便新手去使用，避开了繁杂的编程。

                    # 1.2.2 图形化界面

                      在Stata中，图形化操作均能以命令替代，这样，图形化的操作能够提高人为处理的效率。

                      上图是打开Stata后能看到的界面，在这个界面中，拥有菜单栏、快捷栏，以及历史窗口、结果窗口、命令窗口、变量窗口、属性窗口这五个窗口。
                      命令窗口负责输入你要输入的命令，当前界面下所输入的命令都会留存在历史窗口中。输入命令后所得的结果会返回到结果窗口中（包含当前输入的命令与该命令对应的结果），如果你的命令在Stata中执行错误（也就是出bug了），结果窗口会以红字标注出错。
                      至于变量和属性窗口，这两个会在下一章数据文件与数据的管理细说。

                    # 1.4 案例

                    需求：

                    1. 请将auto数据另存到c:\data下，并以auto2命名；
                    2. 通过help菜单来调出regress命令帮助文件，并简要解释noconstant、vce(robust)参数的作用；
                    3. 请通过查阅edit命令的帮助文件，并说明editprice if price>13000的可能功能。

                    示例：

                    1. 在这里，我们首先使用到了auto.dta数据文件，这是stata自带的示例数据，同时将之保存到了C盘下的data目录，命名为auto2.dta。
                    use auto.dta
save C:\data\auto2.dta
                    1. 使用help帮助命令获取regress命令，也就是回归命令的帮助文件。
                    help regress

                      结果如下：

                      其中，noconstant为不输出常数项，vce(robust)为不输出Source部分
                      下面说明一下两个参数的不同，首先我们来看看无参数输出的情况。

                    regress price mpg rep78 weight

                    regress price mpg rep78 weight ,noconstant

                      使用了noconstant参数，可以看到常数项_CONS不见了

                    regress price mpg rep78 weight ,vce(robust)

                      使用了vce(robust)参数，可以看到来源表格Source不见了

                    1. 同样使用help帮助文件获取有关edit命令的帮助
                    help edit

                      结果如下：

                    关于简写

                    图中命令用下划线标注的为该命令的简写，例如edit简写为ed

                    edit price if price>13000

                      上述命令结果如下：

                      因此可以说明，if是一个条件判断，edit price if price>13000命令的作用为提供价格大于13000的数据以编辑

                      综上，这些命令可以集合成一个dofile，登录Gitee并点击这里可以查看这个dofile的demo，或者查看下面这个简略版的dofile。

                    dofile

regress price mpg rep78 weight ,noconstant
regress price mpg rep78 weight,vce(robust)
edit price if price&gt;13000
